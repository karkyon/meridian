## 2. ホスティングサービス

### 2.1 ホスティングサービス比較

| 項目 | GitHub Pages | Vercel | Netlify |
|------|-------------|--------|---------|
| **料金** | 無料 | 無料（Pro: $20/月） | 無料（Pro: $19/月） |
| **デプロイ方法** | Git Push | Git連携 or CLI | Git連携 or CLI |
| **ビルド機能** | なし（静的のみ） | あり | あり |
| **カスタムドメイン** | 可能 | 可能 | 可能 |
| **HTTPS** | 自動 | 自動 | 自動 |
| **帯域制限** | 100GB/月 | 100GB/月 | 100GB/月 |
| **転送量制限** | ソフトリミット | 100GB/月 | 100GB/月 |
| **プレビュー環境** | なし | あり（PR毎） | あり（PR毎） |
| **Analytics** | なし | 有料 | 有料 |
| **Edge Functions** | なし | あり（有料） | あり（有料） |
| **推奨度** | ★★★★★ | ★★★★☆ | ★★★★☆ |

### 2.2 推奨: GitHub Pages

#### 選定理由
1. **完全無料**：個人・商用利用ともに無料
2. **シンプル**：設定が簡単、Git Pushのみでデプロイ
3. **安定性**：GitHub運用の高い信頼性
4. **HTTPS標準**：Let's Encryptによる自動SSL
5. **Git連携**：リポジトリと完全統合

#### 制約事項
- 静的サイトのみ（サーバサイド処理不可）
- 月間転送量: 100GB（通常利用では十分）
- リポジトリサイズ: 1GB推奨
- ビルド機能なし（Phase2でビルドが必要になったらVercelへ移行検討）

## 3. GitHub Pages環境

### 3.1 セットアップ手順

#### ステップ1: リポジトリ作成
```bash
# GitHubで新規リポジトリ作成
# リポジトリ名: cycleearth-web
# Public / Private どちらでも可（GitHub Pages利用可能）

# ローカルで初期化
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/your-username/cycleearth-web.git
git push -u origin main
```

#### ステップ2: GitHub Pages有効化
```
1. GitHubリポジトリページを開く
2. Settings → Pages を選択
3. Source: Deploy from a branch
4. Branch: main / (root)
5. Save
6. 数分後、デプロイ完了
7. URL: https://your-username.github.io/cycleearth-web/
```

```bash
# リポジトリルートにCNAMEファイル作成

git push

# DNS設定（ドメインレジストラ側）
# Aレコード:
#   185.199.108.153
#   185.199.109.153
#   185.199.110.153
#   185.199.111.153
# または
# CNAMEレコード: your-username.github.io
```


#### 使用状況監視
```
Cesium Ion Dashboard → Usage
- 月間リクエスト数確認
- 無料枠超過前にアラート設定
```

### 5.2 Nominatim API

#### サービス概要
- **提供内容**: OpenStreetMapベースのGeocoding
- **料金**: 完全無料
- **API制限**: 1リクエスト/秒（Usage Policy準拠必須）

#### Usage Policy準拠
```javascript
// GeocodingService.js
class GeocodingService {
  constructor() {
    this.lastRequestTime = 0;
    this.MIN_INTERVAL = 1000;  // 1秒
  }

  async search(query) {
    // レート制限対応
    const now = Date.now();
    const elapsed = now - this.lastRequestTime;
    if (elapsed < this.MIN_INTERVAL) {
      await this.sleep(this.MIN_INTERVAL - elapsed);
    }
    
    this.lastRequestTime = Date.now();
    
    // User-Agent設定（Usage Policy要求）
    const response = await fetch(
      `https://nominatim.openstreetmap.org/search?q=${query}&format=json`,
      {
        headers: {
          'User-Agent': 'CycleEarth/1.0 (https://cycleearth.com)'
        }
      }
    );
    
    return response.json();
  }
}
```

## 8. バックアップ・災害対策

### 8.1 バックアップ戦略

#### コードバックアップ
```
- Git リポジトリ（GitHub）: 自動バックアップ
- ローカル開発環境: 定期プル
- タグ・リリース: バージョン管理
```

#### データバックアップ（Phase1: 不要）
```
- Phase1では全データがローカル（LocalStorage）
- ユーザー側でエクスポート機能を提供（Phase2）
```

### 8.2 災害対策（DR: Disaster Recovery）

#### GitHub Pages障害時
```
1. 代替ホスティングへ切り替え
   - Vercel: 数分でデプロイ可能
   - Netlify: 同上
   
2. DNSレコード変更
   - TTL短縮（事前設定推奨: 300秒）
   - CNAMEレコード変更
```

#### RTO/RPO目標
| 指標 | 目標値 | 説明 |
|------|--------|------|
| RTO（復旧時間目標） | 1時間 | サービス停止から復旧まで |
| RPO（復旧時点目標） | 0分 | データ損失なし（Git管理） |

## 11. 運用手順書

### 11.1 デプロイ手順

#### 通常デプロイ
```bash
# 1. 変更をコミット
git add .
git commit -m "[Release] バージョン1.0.1"

# 2. タグ付け（リリース時）
git tag -a v1.0.1 -m "Version 1.0.1"

# 3. プッシュ
git push origin main
git push origin v1.0.1

# 4. GitHub Pages自動デプロイ（1～3分）
# 5. デプロイ確認
# https://your-username.github.io/cycleearth-web/
```

#### ロールバック
```bash
# 1. 前バージョンのタグを確認
git tag

# 2. 前バージョンをチェックアウト
git checkout v1.0.0

# 3. 強制プッシュ（緊急時のみ）
git push origin main --force

# または、Revert
git revert <commit-hash>
git push origin main
```

### 11.2 障害対応

#### 障害レベル定義
| レベル | 内容 | 対応時間 |
|--------|------|---------|
| P1（緊急） | サービス全停止 | 即時 |
| P2（高） | 主要機能停止 | 1時間以内 |
| P3（中） | 一部機能停止 | 4時間以内 |
| P4（低） | 軽微なバグ | 次回リリース |

#### 障害対応フロー
```
1. 障害検知（監視 or ユーザー報告）
   ↓
2. 影響範囲確認
   ↓
3. 緊急対応（ロールバック等）
   ↓
4. 原因調査
   ↓
5. 恒久対策
   ↓
6. 再発防止策実施
```
