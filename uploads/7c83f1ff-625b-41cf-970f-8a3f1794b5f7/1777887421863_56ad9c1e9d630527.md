### 3.2 各機能の詳細仕様

#### F-20: センサー検索・接続 (BLE-001)

**概要：** BLEケイデンスセンサーをスキャンして接続

**処理フロー：**
```
1. ユーザーが「センサー接続」ボタンをクリック
2. ブラウザがBLEデバイススキャン開始
3. 検出されたセンサー一覧を表示
4. ユーザーがセンサーを選択
5. 接続確立
6. センサーデータ取得開始
```

**技術仕様：**
```javascript
// Web Bluetooth API使用
async function connectSensor() {
  try {
    // BLEデバイス検索（CSC = Cycling Speed and Cadence）
    const device = await navigator.bluetooth.requestDevice({
      filters: [{ services: [0x1816] }],  // CSC Service UUID
      optionalServices: ['battery_service']
    });
    
    // GATTサーバー接続
    const server = await device.gatt.connect();
    
    // CSCサービス取得
    const service = await server.getPrimaryService(0x1816);
    
    // CSC Measurement Characteristic
    const characteristic = await service.getCharacteristic(0x2A5B);
    
    // 通知開始
    await characteristic.startNotifications();
    characteristic.addEventListener('characteristicvaluechanged', handleSensorData);
    
    console.log('センサー接続成功');
  } catch (error) {
    console.error('センサー接続エラー:', error);
  }
}
```

**対応センサー：**
- Bluetooth Low Energy (BLE) 対応
- Cycling Speed and Cadence Service (UUID: 0x1816) 準拠
- 推奨製品：XOSS, CooSpo, Wahoo, Garmin等

#### F-23: キャリブレーション (CAL-001)

**概要：** 初回セットアップ時に「1回転あたりの進行距離」を設定

**キャリブレーション方法：**

**方式1: 実測キャリブレーション（推奨）**
```
1. 地図上に2点（A地点・B地点）を設定
2. A地点からB地点までの直線距離を表示（例：100m）
3. ユーザーがペダルを漕ぎ始める
4. 回転数をカウント
5. B地点に到達したらストップ
6. 総回転数を記録（例：50回転）
7. 係数を計算：100m ÷ 50回転 = 2.0 m/回転
8. 係数を保存
```

**方式2: 簡易キャリブレーション**
```
1. ユーザーに「10回漕いでください」と指示
2. 10回転後の進行距離を確認（例：20m）
3. 係数を計算：20m ÷ 10回転 = 2.0 m/回転
4. 「50回漕いでください」で再確認
5. 係数を微調整
```

**キャリブレーション画面:**

#### F-25: センサー連動移動 (MOV-003)

**概要：** センサーからの回転数データをもとに地球上を移動

**データフロー：**
```
[BLEセンサー]
   ↓ 回転数データ送信（50ms間隔）
[Web Bluetooth API]
   ↓ データ受信
[SensorDataParser]
   ↓ 累積回転数＋時間を取得
[MovementController]
   ↓ 差分回転数を計算
   ↓ 回転数 × 係数 = 移動距離（m）
   ↓ 新座標を計算
[CesiumJS Camera]
   ↓ カメラ位置更新
[地球表示]
```

**実装例：**
```javascript
// センサーデータ受信
function handleSensorData(event) {
  const value = event.target.value;
  
  // BLE CSC Measurementパース
  const flags = value.getUint8(0);
  const hasCrankData = flags & 0x02;
  
  if (hasCrankData) {
    // 累積クランク回転数（2バイト）
    const cumulativeCrankRevolutions = value.getUint16(1, true);
    
    // 最終クランクイベント時間（2バイト、1/1024秒単位）
    const lastCrankEventTime = value.getUint16(3, true);
    
    // 前回データとの差分を計算
    const deltaRevolutions = cumulativeCrankRevolutions - previousRevolutions;
    const deltaTime = (lastCrankEventTime - previousTime) / 1024;  // 秒
    
    // RPM（回転数/分）計算
    const rpm = (deltaRevolutions / deltaTime) * 60;
    
    // 移動距離計算
    const distance = deltaRevolutions * calibrationCoefficient;  // m
    
    // 速度計算
    const speed = (distance / deltaTime) * 3.6;  // km/h
    
    // カメラ移動
    moveCamera(distance, currentHeading);
    
    // 前回データ更新
    previousRevolutions = cumulativeCrankRevolutions;
    previousTime = lastCrankEventTime;
  }
}
```

**カメラ移動計算：**
```javascript
function moveCamera(distanceMeters, heading) {
  // 現在位置取得
  const position = viewer.camera.positionCartographic;
  const lat = Cesium.Math.toDegrees(position.latitude);
  const lon = Cesium.Math.toDegrees(position.longitude);
  const alt = position.height;
  
  // 新座標計算（Haversine formula使用）
  const earthRadius = 6371000;  // m
  const headingRad = Cesium.Math.toRadians(heading);
  
  const newLat = lat + (distanceMeters / earthRadius) * (180 / Math.PI) * Math.cos(headingRad);
  const newLon = lon + (distanceMeters / earthRadius) * (180 / Math.PI) * Math.sin(headingRad) / Math.cos(Cesium.Math.toRadians(lat));
  
  // カメラ移動
  viewer.camera.setView({
    destination: Cesium.Cartesian3.fromDegrees(newLon, newLat, alt),
    orientation: {
      heading: Cesium.Math.toRadians(heading),
      pitch: Cesium.Math.toRadians(-5),
      roll: 0.0
    }
  });
}
```

## 4. センサー連動仕様

### 4.1 BLE通信仕様

#### Cycling Speed and Cadence Service (CSC)
| 項目 | 値 |
|------|---|
| Service UUID | 0x1816 |
| Characteristic UUID | 0x2A5B (CSC Measurement) |
| 通知頻度 | 約20Hz（50ms間隔） |
| データ形式 | バイナリ（詳細は下記） |

#### CSC Measurementデータ構造
```
Byte 0: Flags
  - Bit 0: Wheel Revolution Data Present (0 = No, 1 = Yes)
  - Bit 1: Crank Revolution Data Present (0 = No, 1 = Yes)
  - Bit 2-7: Reserved

※ Phase2ではCrank Revolution Data（Bit 1）のみ使用

Crank Revolution Data Present (Bit 1 = 1) の場合:
Byte 1-2: Cumulative Crank Revolutions (uint16)
Byte 3-4: Last Crank Event Time (uint16, 1/1024秒単位)
```

### 4.2 センサー状態監視

#### 接続状態
| 状態 | 表示 | アクション |
|------|------|-----------|
| 未接続 | 🔴 未接続 | 「接続」ボタン表示 |
| 接続中 | 🟡 接続中... | ローディング表示 |
| 接続済み | 🟢 接続済み | センサー名表示 |
| 切断 | 🔴 切断されました | 再接続ボタン |

#### バッテリー監視（オプション）
```javascript
// Battery Service (UUID: 0x180F)
// Battery Level Characteristic (UUID: 0x2A19)
async function getBatteryLevel(server) {
  try {
    const service = await server.getPrimaryService('battery_service');
    const characteristic = await service.getCharacteristic('battery_level');
    const value = await characteristic.readValue();
    const batteryLevel = value.getUint8(0);
    console.log(`バッテリー: ${batteryLevel}%`);
    return batteryLevel;
  } catch (error) {
    console.log('バッテリー情報取得不可');
    return null;
  }
}
```

## 5. キャリブレーション仕様

### 5.1 キャリブレーション係数

**係数の意味：** 1回転あたりの進行距離（メートル）

**想定範囲：** 1.5m～3.0m（一般的なルームバイク）

**デフォルト値：** 2.0m/回転（未キャリブレーション時）

### 5.2 再キャリブレーション

**タイミング：**
- ユーザーが「設定」から手動実行
- センサー交換時
- ルームバイク変更時

**簡易調整：**
```
「距離が実際より短い」→ 係数を増やす
「距離が実際より長い」→ 係数を減らす

調整UI:
係数: [1.8] [-0.1] [+0.1]
     ←─────●─────→
    1.5m        3.0m
```
