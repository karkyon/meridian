## 1. プロジェクト概要

### 1.1 プロジェクト名
**CycleEarth（サイクルアース）** ※仮称

### 1.2 概要
世界中の任意の場所を指定し、その地点をスタート地点として3D地球上をリアルタイムで「サイクリング視点」で自由に移動できるWebアプリケーション。

### 1.3 Phase1の位置づけ
- センサーなしでの基本動作検証
- CesiumJS連携の技術実証
- UI/UX の初期デザイン確立
- Phase2（BLEセンサー連動）への技術基盤構築

---

## 2. 目的と背景

### 2.1 目的
1. **リアルな地球上でのサイクリング体験** を提供
2. **低コスト** での実現（Google Earth API等の高額サービスを回避）
3. **将来的な拡張性** の確保（VRゴーグル、センサー連動）

### 2.2 背景
- 既存のバーチャルサイクリングアプリ（Zwift、MyWhooshなど）は仮想空間が主体
- 実在のルート映像を使う製品（ROUVY、Kinomapなど）はルートが固定
- **実在の地球を自由に走れる** という新しい体験価値を提供

### 2.3 競合との差別化
| 項目 | 既存サービス | CycleEarth |
|------|------------|------------|
| 走行空間 | 仮想世界 or 固定映像 | 実在の地球（自由移動） |
| 自由度 | ルート固定 | 完全自由 |
| 機器 | 専用スマートバイク推奨 | どんなルームバイクでもOK |
| コスト | 月額課金・高額機器 | 低コスト（センサーのみ） |

### 4.1 機能一覧

| No | 機能ID | 機能名 | 優先度 | 概要 |
|----|--------|--------|--------|------|
| F-01 | LOC-001 | 地名・キーワード検索 | 必須 | 入力したキーワードで該当地点へ移動 |
| F-02 | LOC-002 | スタート位置設定 | 必須 | 検索後の地点をスタート地点として登録 |
| F-03 | CAM-001 | サイクリング視点 | 必須 | 地上約1.7mの高さでカメラ視点を構築 |
| F-04 | MOV-001 | 自動進行モード | 必須 | 一定速度で前進（速度調整可能） |

| F-06 | CAM-002 | 視点操作 | 必須 | マウス or UIボタンで方向転換・視点上下 |
| F-07 | VIS-001 | 地形表示 | 必須 | CesiumJSのTerrain機能で起伏を反映 |
| F-08 | CTL-001 | スピード調整 | 必須 | スライダーで速度調整（1～40km/h） |
| F-09 | INF-001 | 情報オーバーレイ | 必須 | 緯度経度・標高・距離を画面表示 |
| F-10 | REC-001 | ルート記録 | 推奨 | 移動軌跡を線で表示（Polyline） |
| F-11 | CAP-001 | スクリーンショット | 推奨 | 現在の風景を画像保存 |
| F-12 | SET-001 | 設定管理 | 推奨 | 速度・視点高さ・地形ON/OFF設定 |
| F-13 | SAV-001 | データ保存 | オプション | LocalStorageでルート・設定保存 |

### 4.2 各機能の詳細仕様

#### F-01: 地名・キーワード検索 (LOC-001)
**概要：** ユーザーが入力したキーワードをもとに、該当地点へカメラを移動する

**入力：**
- 地名（例：Kyoto、Mount Fuji、Paris）
- 住所（例：東京タワー）
- 座標（例：35.6762, 139.6503）

**処理：**
1. キーワードをGeocoding API（Nominatim等の無料API）で検索
2. 緯度経度を取得
3. Cesiumのカメラを該当地点へアニメーション移動

**出力：**
- カメラが該当地点へ移動
- 地点名・座標を情報パネルに表示

**制約：**
- 検索結果が複数ある場合は最初の候補を採用（将来は候補一覧表示）
- 無効なキーワードの場合はエラーメッセージ表示

---

#### F-03: サイクリング視点 (CAM-001)
**概要：** 地上約1.7m（人の目の高さ）の視点で進行方向を向いたカメラ設定

**パラメータ：**
- 高度：地上 + 1.7m
- 視野角：60～90度（調整可能）
- 向き：進行方向（方位角で管理）

**処理：**
```javascript
viewer.camera.setView({
  destination: Cesium.Cartesian3.fromDegrees(longitude, latitude, altitude + 1.7),
  orientation: {
    heading: Cesium.Math.toRadians(heading), // 方位角
    pitch: Cesium.Math.toRadians(-5),        // やや下向き
    roll: 0.0
  }
});
```

---

#### F-04: 自動進行モード (MOV-001)
**概要：** 設定された速度で自動的に前進

**パラメータ：**
- 速度：1～40 km/h（デフォルト：15 km/h）
- 更新間隔：50ms（20fps）

**計算式：**
```
距離（m） = 速度（km/h） × 1000 / 3600 × 更新間隔（秒）
新緯度 = 現在緯度 + 距離 × cos(方位角) / 111320
新経度 = 現在経度 + 距離 × sin(方位角) / (111320 × cos(現在緯度))
```

**制御：**
- スタート/ストップボタンで制御
- スペースキーでも制御可能

---

#### F-06: 視点操作 (CAM-002)
**概要：** マウスまたはUIボタンで進行方向・視点を変更

**操作方法：**
1. マウスドラッグ：視点の左右・上下
2. 矢印キー：方向転換（左右）
3. UIボタン：方向転換ボタン（画面右下）

**方向転換の仕様：**
- 1回の操作で15度回転
- スムーズなアニメーション（0.5秒）

---

#### F-07: 地形表示 (VIS-001)
**概要：** 実際の地形の起伏を反映

**使用データ：**
- Cesium World Terrain（無料）
- 解像度：最大15m/pixel

**設定：**
```javascript
viewer.terrainProvider = Cesium.createWorldTerrain({
  requestWaterMask: true,  // 海の表現
  requestVertexNormals: true  // 陰影表現
});
```

---

#### F-09: 情報オーバーレイ (INF-001)
**概要：** 現在地情報を画面に表示

**表示項目：**
- 緯度（Latitude）
- 経度（Longitude）
- 標高（Altitude）：地形高度 + カメラ高度
- 総移動距離（Total Distance）
- 現在速度（Current Speed）
- 進行方位（Heading）

**表示位置：** 画面左下

**更新頻度：** 毎フレーム（リアルタイム）

---

#### F-10: ルート記録 (REC-001)
**概要：** 移動した軌跡を線で表示

**実装方法：**
```javascript
viewer.entities.add({
  polyline: {
    positions: Cesium.Cartesian3.fromDegreesArray([
      // 移動した座標の配列
    ]),
    width: 3,
    material: Cesium.Color.CYAN
  }
});
```

**仕様：**
- 線の色：シアン（#00FFFF）
- 線の太さ：3ピクセル
- 記録間隔：1秒ごと

## 5. 非機能要件

### 5.1 性能要件
| 項目 | 要件 | 備考 |
|------|------|------|
| 初回読込時間 | 5秒以内 | 3G回線環境 |
| カメラ移動のフレームレート | 30fps以上 | 滑らかな移動体験 |
| 検索応答時間 | 2秒以内 | Geocoding APIのレスポンス含む |
| メモリ使用量 | 500MB以下 | ブラウザメモリ |

### 5.2 互換性要件
| ブラウザ | バージョン | 備考 |
|----------|-----------|------|
| Google Chrome | 最新版～2世代前 | 推奨 |
| Microsoft Edge | 最新版～2世代前 | 推奨 |
| Safari | 最新版～2世代前 | 一部機能制限あり |
| Firefox | 最新版～2世代前 | 対応 |

**必須要件：**
- WebGL 2.0対応
- JavaScript ES6対応

### 5.3 ユーザビリティ要件
- 直感的な操作（5分以内に基本操作を習得可能）

## 6. 制約事項

### 6.1 技術的制約
1. **API制限**
   - Cesium Ion無料枠：月間1,000リクエストまで
   - Geocoding API：1日あたり2,500リクエストまで（Nominatim）

2. **ブラウザ制約**
   - WebGL未対応ブラウザでは動作不可
   - モバイル端末では性能低下の可能性

3. **地形データ制約**
   - 詳細な建物3Dモデルは限定的（大都市のみ）
   - 一部地域でテクスチャ解像度が低い

### 6.2 Phase1での機能制限
- **センサー連動なし**（Phase2で対応）
- **VRゴーグル非対応**（Phase3で対応）
- **マルチユーザー機能なし**（Phase4で対応）

## 7. 想定ユーザー

### 7.1 ターゲットユーザー
1. **フィットネス愛好者**
   - ルームバイクで運動する人
   - 飽きずに続けたい人

2. **旅行・地理好き**
   - 世界各地を仮想的に走りたい
   - Google Earthのような体験を求める

3. **テクノロジー愛好者**
   - 新しい体験を試したい
   - VR/AR技術に興味がある

### 7.2 ユーザーストーリー

**ストーリー1：世界旅行気分**
> ユーザーAさんは、ルームバイクで運動中に「パリのエッフェル塔周辺を走りたい」と思い、検索ボックスに「Eiffel Tower」と入力。カメラがパリへ移動し、エッフェル塔を眺めながらセーヌ川沿いをサイクリング。

**ストーリー2：故郷を走る**
> ユーザーBさんは、故郷の京都を懐かしみながら運動したいと考え、「京都 清水寺」と検索。清水寺からスタートし、東山エリアを自由に走る。

**ストーリー3：フィットネスの記録**
> ユーザーCさんは、毎日の運動記録を残したいと考え、ルート記録機能をONにして走行。画面に表示される総移動距離を見ながらモチベーションを維持。

## 10. Phase2への展開

### 10.1 Phase2の概要
**テーマ：** BLEケイデンスセンサー連動

**主な追加機能：**
1. BLE通信実装（Web Bluetooth API）
2. センサーからの回転数取得
3. 回転数→距離・速度換算
4. 初期キャリブレーション機能
5. リアルタイム連動

### 10.2 Phase2への技術引き継ぎ
| 項目 | Phase1 | Phase2 |

| 移動制御 | 自動・手動（UI） | センサー連動 |
| 速度 | UI設定 | センサー計算 |
| 距離 | 時間ベース | 回転数ベース |

### 10.3 Phase2スケジュール（予定）
- 期間：4週間
- 開始：Phase1完了後2週間以内
